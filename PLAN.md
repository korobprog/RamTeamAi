# Neurogate — план разработки

Настольный AI-клиент на Tauri. Подключение **любых** ИИ-API (включая кастомные),
оркестрация нескольких моделей как **команды суб-агентов** и **Planning Mode**, где
модели обсуждают задачу, ищут в интернете/MCP и выдают редактируемое решение,
которое разворачивается в реальный проект.

> Имя «Neurogate» рабочее — можно переименовать.

---

## 1. Ключевые фичи

1. **Universal API Connector** — подключение любого AI API: Anthropic, OpenAI,
   Google Gemini, OpenAI-совместимые (Groq/Together/OpenRouter), локальные
   (Ollama/LM Studio) и **полностью кастомные эндпоинты** через маппинг полей.
2. **Multi-Agent Mode** — несколько агентов (можно с **разных** API и разных
   моделей) работают вместе как суб-агенты: supervisor / debate / pipeline.
3. **Planning Mode → Project Scaffolding** — команда агентов обсуждает задачу,
   ищет в вебе и MCP, сходится к решению; пользователь правит артефакт и жмёт
   **Build** — структура проекта создаётся на диске.

---

## 2. Технологический стек

| Слой | Технология | Зачем |
|------|-----------|-------|
| Оболочка | Tauri 2.x (Rust core) | лёгкий бинарь, нативная безопасность, доступ к ФС/процессам |
| Frontend | React + TypeScript + Vite | богатый UI |
| UI-kit | shadcn/ui + Tailwind | быстрый кастомный дизайн |
| Стейт | Zustand | сессии/агенты |
| Backend-логика | Rust (Tauri commands) | оркестрация, потоки, безопасные ключи |
| Хранилище | SQLite (`tauri-plugin-sql` / `sqlx`) | история, конфиги, проекты |
| Секреты | OS keychain (`keyring` / `tauri-plugin-stronghold`) | шифрованное хранение ключей |
| Стриминг | SSE / WebSocket в Rust | потоковый вывод токенов |

---

## 3. Архитектура ядра

```
Frontend (React): Chat UI | Agent Builder | Planning Board | Project View
        │  Tauri IPC (commands/events)
Rust Core:
  Provider Adapters  — единый интерфейс к любому API (+ кастомный маппинг)
  Orchestrator       — мультиагентная логика (supervisor/debate/pipeline)
  MCP Manager        — запуск MCP-серверов, реестр инструментов, web search
  Secrets Vault      — ключи в OS keychain
  Storage (SQLite)   — сессии, агенты, команды, планы, проекты
  Project Builder    — разворачивание структуры проекта на ФС
```

### Provider Adapters
Абстракция «провайдер» с унифицированным запросом. Кастомный адаптер задаётся в UI:
base URL, тип аутентификации (header/bearer/query), шаблон тела запроса
(маппинг `{{model}}`, `{{messages}}`, `{{stream}}`…), JSONPath к ответу и к
потоковым чанкам, формат стриминга. У каждой модели — capability-флаги
(streaming, tool-use, vision, max context), которые учитывает оркестратор.

### Orchestrator
`Agent` = {провайдер + модель + системный промпт + роль + инструменты}.
Нормализует общий диалог под формат каждого провайдера (особое внимание —
tool-use и стримингу). Топологии: supervisor → sub-agents; peer debate;
pipeline. Общий context bus с метками авторов. Лимиты раундов/токенов + арбитр
против зацикливания.

### MCP Manager
Запуск MCP-серверов (stdio / HTTP), реестр инструментов, доступных любому
агенту. Интернет-поиск — через MCP-сервер поиска или встроенный web-search tool.

> В `prototype.html` верхняя навигация переключает экраны; кнопки внутри экранов
> воспроизводят реальные переходы (онбординг → провайдеры → кастомный API →
> агент → топология → диалог → решение → Build).

### Дизайн-язык
- Плоский, чистый, без градиентов/теней. Минимальные бордеры `0.5px`.
- Радиусы: `--border-radius-md` (8px) элементы, `--border-radius-lg` (12px) карточки.
- Типографика: заголовки 18px/500, тело 13–16px/400, два веса — 400 и 500.
- Регистр предложений (не Title Case, не CAPS). Иконки — Tabler outline.
- Тёмная тема обязательна; все цвета через CSS-переменные.

### Цветовое кодирование агентов (сквозное)
- **Архитектор** — фиолетовый (`#534AB7`, фон-метка `#EEEDFE`)
- **Критик** — зелёный/teal (`#1D9E75`, текст `#0F6E56`)
- **Исследователь** — коралловый (`#D85A30`, текст `#993C1D`)
- Статусы: success — соединение/активен, warning — ключ истёк, info — выбор/MCP.

### Экраны (спецификация)

1. **Онбординг** — приветствие + 3 кликабельных шага: подключить API (✓) →
   собрать команду (текущий, рамка info 2px) → запустить Planning Mode.
   Внизу — индикатор прогресса (3 точки).

2. **Настройки провайдеров** — список карточек: иконка-плашка, имя, замаскированный
   ключ (моно-шрифт), число моделей, статус (success/warning), меню `⋯`.
   Кнопка «Подключить API». Снизу — заметка про keychain.

3. **Мастер кастомного API** — поля: Название, Base URL, Аутентификация (select),
   Стриминг (select), textarea маппинга тела (моно), JSONPath ответа и чанка,
   чипы capability (streaming/tool-use/vision/ctx). Низ: статус теста соединения
   + кнопки «Тест» / «Сохранить».

4. **Agent Builder** — аватар-кружок роли, имя (input), сетка 2×2: Провайдер /
   Модель / Роль / Бюджет токенов; textarea системного промпта; чипы инструментов
   (Web search / Файлы / MCP) + «добавить»; кнопки «Отмена» / «Сохранить агента».

5. **Выбор топологии** — 3 карточки с мини-SVG-схемами: Supervisor (звезда вниз),
   Debate (выбрана, рамка info 2px, бейдж «Planning»), Pipeline (стрелки →).
   Низ: «Макс. раундов», «Арбитр», кнопка «Запустить команду».

6. **Мультиагентный диалог** (главный экран) — 3 колонки: сайдбар (сессии/команды),
   центр (переключатель Planning/Chat, лента реплик с левым цветным бордером по
   автору, поле ввода), правая панель активных агентов со статусами
   (пишет / ждёт / MCP). Счётчик токенов в шапке.

7. **Решение → Build** — слева редактируемый план (стек, шаги с drag-хэндлами,
   подсветка правки, «Добавить шаг»), справа дерево будущего проекта (моно) и
   кнопка «Build проект» (заливка info).

## 5. Данные и безопасность

- **SQLite:** сессии, сообщения, агенты, пресеты команд, история планов, проекты.
- **Vault:** API-ключи в OS keychain; в БД — только ссылки.
- **Project Builder:** запись на ФС и запуск кода — только через подтверждение
  пользователя и песочницу.

## 7. Главные технические риски

1. **Унификация форматов** разных API (tool-use, стриминг) — заложить абстракцию сразу.
2. **Зацикливание агентов** в дебатах — лимиты раундов/токенов + арбитр.
3. **Стоимость** мультиагентных прогонов — счётчик токенов и бюджет на сессию.
4. **Безопасность Project Builder** — подтверждение и песочница для ФС/запуска кода.
